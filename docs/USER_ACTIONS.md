# Подключение аккаунтов — пошагово (для человека, не программиста)

Почти всё делает мастер. От тебя нужен только **один разовый шаг на стороне
платформы** — создать «приложение» (это требует твоего аккаунта, поэтому за тебя
это никто сделать не может). Дальше просто вставляешь два значения в мастер, жмёшь
кнопку — и всё.

## Сначала открой мастер

В терминале (один раз):

```bash
cd ~/Projects/АГЕНТСКИЙ_БРОКЕР/mcp-server
source .venv/bin/activate
broker-wizard
```

Откроется страница в браузере (`http://127.0.0.1:8765`). Там выбираешь платформу и
идёшь по шагам. Ниже — тот же путь, но с подробностями и картинкой «что нажимать»
на стороне платформы. **Мастер сам создаёт настройку в Nango, сам ловит
подключение и сам всё сохраняет** — тебе не нужно ни редактировать файлы, ни
заходить в панель Nango.

> ⚠️ **Золотое правило.** На шаге создания приложения будет поле «Redirect URI»
> (адрес возврата). Туда нужно вставить **ровно** этот адрес, без единой опечатки:
>
> ```
> http://localhost:3003/oauth/callback
> ```
>
> Мастер показывает его с кнопкой «Копировать» — копируй оттуда, не набирай руками.
> Одна лишняя буква → ошибка `redirect_uri_mismatch` при входе.

## 1. YouTube — самый простой, начни с него (~10 минут)

Тебе нужен обычный аккаунт Google с каналом на YouTube.

### Часть А. Создать приложение в Google (это и есть твой разовый шаг)

1. Открой **https://console.cloud.google.com** и войди своим Google-аккаунтом.
2. **Создай проект** (это просто «папка» для настроек):
   - Вверху слева, рядом с надписью «Google Cloud», нажми на **выпадающий список
     проектов** → **New Project / Новый проект**.
   - Имя: например `content-broker`. Нажми **Create / Создать**. Подожди пару
     секунд и убедись, что вверху выбран именно этот проект.
3. **Включи YouTube API:**
   - В строке поиска вверху набери **YouTube Data API v3** → выбери его в
     результатах → нажми синюю кнопку **Enable / Включить**.
4. **Настрой «экран согласия» (Google Auth Platform):**
   - Слева меню → **APIs & Services → OAuth consent screen** (в новом интерфейсе
     этот раздел называется **Google Auth Platform**).
   - Если видишь «Google Auth Platform not configured» — нажми **Get started /
     Начать**. Пройди 4 коротких шага:
     - **App Information:** App name = любое (напр. `Content Broker`); User
       support email = твоя почта из списка.
     - **Audience / Тип пользователя:** выбери **External / Внешние**.
     - **Contact Information:** твоя почта.
     - Согласись с условиями → **Create / Создать**.
5. **Добавь себя в тест-пользователи** (иначе Google не пустит на вход, пока
   приложение не опубликовано — но публикация нам не нужна):
   - В **Google Auth Platform → Audience / Аудитория** найди блок **Test users /
     Тестовые пользователи** → **Add users / Добавить** → впиши свой Gmail →
     **Save**.
6. **Создай сам ключ (OAuth client):**
   - **Google Auth Platform → Clients / Клиенты** → **Create client / Создать
     клиента** (или «+ Create credentials → OAuth client ID»).
   - **Application type / Тип приложения:** **Web application / Веб-приложение**.
   - **Name / Имя:** любое, напр. `broker-web`.
   - **Authorized redirect URIs / Разрешённые URI перенаправления** → **Add URI /
     Добавить URI** → вставь скопированный из мастера адрес:
     `http://localhost:3003/oauth/callback`
   - Нажми **Create / Создать**.
7. Появится окно с **Client ID** и **Client secret** — вот эти два значения и
   нужны мастеру. (Их всегда можно посмотреть снова: Clients → твой клиент.)

### Часть Б. В мастере (30 секунд)

1. В окне мастера выбери **YouTube**.
2. Вставь **Client ID** и **Client Secret** из шага А.7 → **Создать интеграцию**.
3. Нажми **Подключить аккаунт YouTube** → в открывшемся окне войди Google-аккаунтом
   и нажми **Разрешить**.
   - Если Google покажет «Google hasn't verified this app» — это нормально для
     тест-режима: нажми **Advanced / Дополнительно → Go to … (unsafe) / Перейти**.
4. Мастер сам увидит подключение и напишет «Готово ✅». Всё.

Теперь агент умеет `youtube_upload` / `youtube_list_videos` для этого канала.

## 2. Instagram — сложнее (нужен бизнес-аккаунт и проверка Meta)

Требуется: аккаунт Instagram типа **«Бизнес»**, привязанный к странице Facebook.
Публикация постов у Meta включается только после **App Review** — заложи на это
время (может занять дни).

1. Открой **https://developers.facebook.com/apps/** → **Create App / Создать
   приложение** → тип **Business / Бизнес**.
2. В приложении добавь продукты **Instagram** (Graph API) и при необходимости
   **Facebook Login**.
3. В настройках входа найди **Valid OAuth Redirect URIs** и вставь тот же адрес:
   `http://localhost:3003/oauth/callback`
4. Права (разрешения), которые понадобятся под публикацию:
   `instagram_business_basic`, `instagram_business_content_publish` — их нужно
   заявить и провести **App Review**.
5. Возьми **App ID** (= Client ID) и **App Secret** (= Client Secret).
6. В мастере выбери **Instagram** → вставь эти два значения → **Создать
   интеграцию** → **Подключить**.
7. При постинге агенту нужно передавать `ig_user_id` — числовой id твоего
   Instagram-бизнес-аккаунта (его видно в Meta или через `instagram_list_media`).

> Пока App Review не пройден, подключение может работать в тестовом режиме только
> для аккаунтов-разработчиков/тестировщиков, добавленных в приложении.

## 3. ВКонтакте — готового провайдера в Nango нет (generic OAuth2)

VK заводится как «generic OAuth2». Мастер уже подставляет адреса VK ID, тебе нужно
только приложение VK:

1. Открой **https://id.vk.com/about/business/go** (VK ID для бизнеса) или
   **https://dev.vk.com** → создай приложение (тип «Веб» / «Сайт»).
2. Redirect URI: `http://localhost:3003/oauth/callback`
3. Права: `wall` (посты на стену), `offline` (долгий доступ).
4. Возьми **ID приложения** и **Защищённый ключ** (Client ID / Secret).
5. В мастере выбери **ВКонтакте** → вставь значения → **Создать интеграцию** →
   **Подключить**.

> Если у этой сборки Nango generic-OAuth2 для VK не заведётся автоматически —
> сообщи мне, я допишу провайдера VK в конфиг Nango (это делается один раз).

## 4. Яндекс Дзен — у него нет API, вход через браузер

Дзен не даёт токенов вообще. Подключение — разовый ручной вход, сохраняется сессия
браузера (cookies), не токен, и живёт отдельно от Nango.

```bash
cd ~/Projects/АГЕНТСКИЙ_БРОКЕР/mcp-server
source .venv/bin/activate
uv pip install patchright && patchright install chromium   # один раз
python scripts/capture_dzen_session.py                      # войди руками, сессия сохранится
```

После этого работают `dzen_list_articles` / `dzen_publish_article`.

## Если что-то пошло не так

| Симптом | Что это и что делать |
|---|---|
| `redirect_uri_mismatch` при входе | Адрес возврата в приложении не совпал. Скопируй из мастера ещё раз **ровно** `http://localhost:3003/oauth/callback` и вставь в Redirect URI приложения. |
| «Google hasn't verified this app» | Нормально для тест-режима. Advanced → Go to … Проверь, что твой Gmail добавлен в Test users. |
| `access_denied` | Ты не в списке Test users, либо не нажал «Разрешить». Добавь себя в тест-пользователи (шаг 1.А.5). |
| Мастер не видит подключение (крутится «жду») | Убедись, что окно входа закрылось успехом. Обнови статус — мастер опрашивает каждые 3 сек. Если долго — проверь, что интеграция создалась (шаг «Создать интеграцию» был зелёным). |
| «Integration does not exist» | Сначала жми **Создать интеграцию** (вставив Client ID/Secret), и только потом **Подключить**. |

## Для друга на его компьютере
Адрес `localhost` работает только на твоей машине. Чтобы подключал друг со своего
компьютера — либо он делает это у тебя, либо нужно временно открыть твой Nango
наружу (туннель) и в приложении добавить публичный Redirect URI. Скажи мне — настрою.

## Где взято «актуально»
Флоу Google приведён под новый интерфейс **Google Auth Platform** (реорганизация
2025–2026: разделы Branding / Audience / Data Access / Clients). Callback-адрес и
права (scopes) берутся из одного источника — `mcp-server/src/broker_connectors/platforms.py`,
чтобы мастер и эта инструкция никогда не расходились.
